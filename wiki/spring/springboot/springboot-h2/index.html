SpringBoot 整合 H2数据库

H2的特点​

  • 非常快，开源，支持JDBC API，占用空间小，jar只有2MB大小
  • 嵌入式和服务器模式；内存数据库

H2支持的连接模式​

1）嵌入式模式（使用 JDBC 的本地连接） 2）服务器模式（使用 JDBC 或 ODBC 在 TCP/IP 上的远程连接） 3）混合模式（本地和远程连接同时进行）

三种模式详解见： https://www.h2database.com/html/features.html#connection_modes https://www.cnblogs.com/cnjavahome/p/8995650.html

其中最常使用的是嵌入式模式，与所有模式一样，支持持久性和内存数据库。

SpringBoot 中使用H2​

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    <dependency>
        <groupId>com.h2database</groupId>
        <artifactId>h2</artifactId>
        <scope>runtime</scope>
    </dependency>
</dependencies>

配置

内存模式，数据仅保存在内存中，当应用程序关闭时内存中的数据会丢失 application-inner.yml

server:
  port: 8080

spring:
  datasource:
    driverClassName: org.h2.Driver
    password: sa
    # 内存
    url: jdbc:h2:mem:dbtest
    # 持久化到文件
    # url: jdbc:h2:file:D:/opt/H2/dbtest
    username: sa
  sql:
    init:
      platform: h2
      # 初始化数据表 DDL
      schema-locations: classpath:db/schema.sql
      # 初始化数据 DML
      data-locations: classpath:db/data.sql
  h2:
    console:
      enabled: true
      # h2控制台访问路径
      path: /h2-console
      settings:
        # 允许console远程访问
        web-allow-others: true
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    open-in-view: false
url配置详解

spring.datasource.url：jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;MODE=MYSQL

  • DB_CLOSE_DELAY：要求最后一个正在连接的连接断开后，不要关闭数据库
  • MODE=MySQL：兼容模式，H2兼容多种数据库，该值可以为：DB2、Derby、HSQLDB、SQLServer、MySQL、Oracle、PostgreSQL
  • AUTO_RECONNECT=TRUE：连接丢失后自动重新连接
  • AUTO_SERVER=TRUE：启动自动混合模式，允许开启多个连接，该参数不支持在内存中运行模式
  • TRACE_LEVEL_SYSTEM_OUT、TRACE_LEVEL_FILE：输出跟踪日志到控制台或文件， 取值0为OFF，1为ERROR（默认值），2为INFO，3为DEBUG
  • SET TRACE_MAX_FILE_SIZE mb：设置跟踪日志文件的大小，默认为16M

访问 http://localhost:8080/users/add?userId=2&userName=Yoyo 添加用户信息，然后再查询用户列表 http://localhost:8080/users/list，发现内存中的用户已经新增。

访问H2控制台：http://localhost:8080/h2-console。注意JDBC URL的值与配置文件中的保持一致，如果没有设置密码直接点connect

持久化模式，修改配置文件的中的 spring.datasource.url=jdbc:h2:file:D:/opt/H2/dbtest

在指定的目录下会生成一个名为 dbtest.mv.db 的数据库文件，

通过console观察，在应用程序关闭后，数据不会丢失，数据已经持久化到本地文件

注意点​

  1. MySQL有些语法在H2中可能不支持，如索引，sql脚本中不能有, 否则 H2 会报错。这些在初始化sql文件都需要替换掉
