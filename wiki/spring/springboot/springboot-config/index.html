SpringBoot配置文件

本文主要介绍读取配置文件的几种常用方式，复杂结构的配置，SpringBoot中配置文件的加载顺序，以及默认情况下的配置信息，以便在项目实践中快速实现。

1. 读取配置的几种方式​

一般来说，配置文件的位置在 src/main/resources 目录下的 application.yml 或 application.properties 文件里。常用有三种方式

1、Environment 读取

2、@Value 注解方式

危险
  • @Value 中指定默认值时不需要空格，也不需要引号

3、配置对象映射方式

AppConfig配置类

@Data
@Configuration
@ConfigurationProperties(prefix = "demo.app")
public class AppConfig {

    private Long id;

    private String key;

}

演示

    @Value("${spring.application.name:default}")
    private String applicationName;
    @Autowired
    private Environment environment;
    @Autowired
    private AppConfig appConfig;

    @RequestMapping("/index")
    public String index() {
        // env中读取配置
        String appId = environment.getProperty("demo.app.id");
        String appKey = environment.getProperty("demo.app.key");
        log.info("properties from env, appId=[{}], appKey=[{}]",
                appId, appKey);

        // 配置对象映射
        log.info("properties from appConfig: [{}]", appConfig);

        // @Value注解获取配置
        return "app-name: " + applicationName;
    }

2. 多环境的配置​

SpringBoot是支持多环境的配置。在目录 src/main/resources 下添加不同配置文件，如 application-dev.yml、application-test.yml、application-prod.yml。

在命令行中对应的环境，通过 spring.profiles.active 来指定，当指定多个时，右边的优先级高于左边。

java -jar ./app.jar --spring.profiles.active=dev

3. 各种复杂结构的配置映射对象​

参考：SpringBoot Externalized Configuration

1) 如何在 application.yml 中配置List、Map

demo:
  app:
    id: 8888
    key: ${random.uuid}
  complex:
    names: ["Hans", "YoYo"]
    userList:
      - name: Hans
        age: 1
      - name: Yoyo
        age: 2
    userMap:
      admin:
        name: Jack
        age: 1
      user:
        name: Coco
        age: 2
    mapList:
      hobbies: ["Football","Running"]
      books: ["活着","局外人"]

对应的配置对象如下

@Data
@Configuration
@ConfigurationProperties("demo.complex")
public class ComplexConfig {

    private List<String> names;

    private List<UserDTO> userList;

    private Map<String, UserDTO> userMap;

    private Map<String, List<String>> mapList;

}

2) 使用注解 @PropertySource 读取外部文件配置

外部文件 resources/config/hk.properties

name=hk
sex=1

对应的配置对象 HKConfig

@Component
@PropertySource(value = "classpath:config/hk.properties")
public class HKConfig {
    @Value("${name}")
    private String name;
    @Value("${sex}")
    private String sex;
    public String getName() {
        return name;
    }
    public void setName(String name) {
        this.name = name;
    }
    public String getSex() {
        return sex;
    }
    public void setSex(String sex) {
        this.sex = sex;
    }
}

4. 配置文件的加载顺序​

SpringBoot官方文档说明了加载的顺序如下，越靠前优先级越高。

Spring Boot uses a very particular PropertySource order that is designed to allow sensible overriding of values. Properties are considered in the following order


1. 开发者工具 Devtools 全局配置参数；  

2. 单元测试上的 [@TestPropertySource](mailto:@TestPropertySource)` 注解指定的参数;  

3. 单元测试上的 [@SpringBootTest](mailto:@SpringBootTest)` 注解指定的参数；  

4. 命令行指定的参数，如 java -jar springboot.jar --name="xxx"；  

5. 命令行中的 SPRING_APPLICATION_JSON 指定参数, 如 java -Dspring.application.json='{"name":"xxx"}' -jar springboot.jar  

6. ServletConfig初始化参数；  

7. ServletContext初始化参数；  

8. JNDI参数（如 java:comp/env/spring.application.json）；  

9. Java系统参数（来源：System.getProperties()）；  

10. 操作系统环境变量参数；  

11. RandomValuePropertySource 随机数，仅匹配：ramdom.*；  

12. JAR包外面的配置文件参数（application-{profile}.properties（YAML））  

13. JAR包里面的配置文件参数（application-{profile}.properties（YAML））  

14. JAR包外面的配置文件参数（application.properties（YAML））  

15. JAR包里面的配置文件参数（application.properties（YAML））  

16. @Configuration (mailto:@Configuration)配置文件上 @PropertySource(mailto:@PropertySource) 注解加载的参数；  

17. 默认参数（通过 SpringApplication.setDefaultProperties 指定）；  

. project-sample  
├── config  
│   ├── application.yml （4）  
│   └── src/main/resources  
|   │   ├── application.yml （1）  
|   │   └── config  
|   |   │   ├── application.yml （2）  
├── application.yml （3）  
  
注：src/main/resources下的配置文件在项目编译时，会放在target/classes下

启动时加载配置文件顺序：1 -> 2 -> 3 -> 4，优先级 4 > 3 > 2 > 1

6. 读取 resources 目录下的文件​

public static void main(String[] args) throws IOException {
    // 方法一
    ClassPathResource classPathResource = new ClassPathResource("test.txt");
    InputStream inputStream = classPathResource.getInputStream();
    // 读取流，如果转为file，打成jar就会失败
    String content = new BufferedReader(new InputStreamReader(inputStream, StandardCharsets.UTF_8))
            .lines().collect(Collectors.joining("\t"));
    log.debug("content: {}", content);
    
    // 方法二
    InputStream resourceAsStream = ImageIOUtils.class.getClassLoader().getResourceAsStream("test.txt");
    String strings = new BufferedReader(new InputStreamReader(resourceAsStream, StandardCharsets.UTF_8))
            .lines().collect(Collectors.joining("\t"));
    log.debug("strings: {}", strings);

    // 读写图片
    BufferedImage bufferedImage = ImageIO.read(new ClassPathResource("imgs/test.jpg").getInputStream());
    // {project}/target/classes, jar包中会报错
    String resourcePath = ImageIOUtils.class.getClassLoader().getResource("").getPath();
    ImageIO.write(bufferedImage, "png", new File(resourcePath + "imgs/123.png"));
}
