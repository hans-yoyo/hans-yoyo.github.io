SpringBoot实现定时任务的几种方式

介绍SpringBoot中实现定时任务的几种方式，有以下几种

  1. 使用JDK自带的Timer, 不推荐
  2. 使用 ScheduledExecutorService，基于线程池设计的定时任务类，不推荐
  3. 使用SpringBoot自带的 @Scheduled 注解实现定时任务，推荐
  4. 使用第三方依赖，如 Quartz
  5. 分布式定时任务 // TODO

Timer实现定时任务​

TimerTask timerTask = new TimerTask() {
    @Override
    public void run() {
        log.info("jdk timer task executed...");
    }
};
Timer timer = new Timer();
// 延迟1s，每3s执行一次
timer.schedule(timerTask, 1000, 3000);

使用 ScheduledExecutorService​

public static void main(String[] args) {
    ScheduledExecutorService scheduledExecutorService = Executors.newScheduledThreadPool(1);
    scheduledExecutorService.scheduleAtFixedRate(() -> log.info("scheduledExecutorService task executed..."), 1000, 3000, TimeUnit.MILLISECONDS);
}

基于SpringBoot自带的注解的方式​

@EnableScheduling 开启对定时任务的支持。可以放在启动类上(推荐)，也可以放在注入定时任务的类上

相关api包含在spring-context的包org.springframework.scheduling下。只需引入下面的依赖

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
@EnableScheduling
@SpringBootApplication
public class TimedTaskApp {

    public static void main(String[] args) {
        SpringApplication.run(TimedTaskApp.class, args);
    }

}

@Slf4j
@Component
public class AnnotationTaskConfig {

    /**
     * 每5s执行一次
     * 还有其他配置：fixedDelay、fixedRate、initialDelay
     */
    @Scheduled(cron = "*/5 * * * * ?")
    public void cronTask() {
        log.info("cronTask executed......");
    }

}

cron解释​

结构：cron表达式是一个字符串，分为6或7个域，每两个域之间用空格分隔。其语法格式为："秒域 分域 时域 日域 月域 周域 年域"

取值范围

# 每隔5秒钟执行一次
*/5 * * * * ?
# 每隔1分钟执行一次
0 * /1 * * * ?
# 每天23点55分执行一次 　
0 55 23 * * ?
# 从第1s开始，每6秒执行一次
1/6 * * * * ?
# 在24分，30分执行一次
0 24,30 * * * ?

基于接口的方式​

有时候我们希望执行计划能够存储在数据库中，当我们需要修改执行计划时，只需要修改数据库就可以，而不用重启服务。

实现核心代码

@Slf4j
@Component
public class DynamicScheduleTask implements SchedulingConfigurer {

    @Autowired
    private ScheduleTaskRepository scheduleTaskRepository;

    @Override
    public void configureTasks(ScheduledTaskRegistrar scheduledTaskRegistrar) {
        scheduledTaskRegistrar.addTriggerTask(() -> doProcess(),
                triggerContext -> {
                    Optional<ScheduleTask> opt = scheduleTaskRepository.findById(1L);
                    if (!opt.isPresent()) {
                        throw new RuntimeException("定时任务不存在");
                    }
                    return new CronTrigger(opt.get().getCron()).nextExecutionTime(triggerContext);
                });
    }

    private void doProcess() {
        log.info("基于接口实现的定时任务");
    }

}

使用第三方依赖，Quartz​

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-quartz</artifactId>
</dependency>
@Slf4j
public class TestQuartz extends QuartzJobBean {
    @Override
    protected void executeInternal(JobExecutionContext jobExecutionContext) throws JobExecutionException {
        log.info("quartz task executed，10s一次");
    }
}

@Configuration
public class QuartzConfig {

    @Bean
    public JobDetail teatQuartzDetail() {
        return JobBuilder.newJob(TestQuartz.class).withIdentity("testQuartz").storeDurably().build();
    }

    @Bean
    public Trigger testQuartzTrigger() {
        SimpleScheduleBuilder scheduleBuilder = SimpleScheduleBuilder.simpleSchedule()
                // 设置时间周期
                .withIntervalInSeconds(10)
                .repeatForever();
        return TriggerBuilder.newTrigger().forJob(teatQuartzDetail())
                .withIdentity("testQuartz")
                .withSchedule(scheduleBuilder)
                .build();
    }

}
