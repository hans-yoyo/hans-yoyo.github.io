SpringBoot 整合 Knife4j

SpringBoot集成Knife4j​

引入依赖​

    <properties>
        <knife4j.version>2.0.9</knife4j.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!--引入Knife4j的官方start包-->
        <dependency>
            <groupId>com.github.xiaoymin</groupId>
            <artifactId>knife4j-spring-boot-starter</artifactId>
            <version>${knife4j.version}</version>
        </dependency>
    </dependencies>

添加Knife4j配置类​

可通过配置 swagger.enabled 启用/关闭 api文档，

/**
 * @author huhan
 */
@Configuration
@EnableSwagger2WebMvc
@ConditionalOnProperty(name = {"swagger.enabled"}, havingValue = "true")
public class Knife4jConfiguration {

    /**
     * 扫描路径
     */
    @Value("${swagger.base-package:com.hans.tech}")
    private String basePackage;
    /**
     * 文档标题
     */
    @Value("${swagger.title:}")
    private String title;
    /**
     * 文档描述
     */
    @Value("${swagger.description:}")
    private String description;
    /**
     * 文档版本号
     */
    @Value("${swagger.version:1.0.0}")
    private String version;

    @Bean
    public Docket customDocket() {
        //指定使用Swagger2规范
        return new Docket(DocumentationType.SWAGGER_2)
                .apiInfo(apiInfo())
                //分组名称
                .groupName("VHR Project")
                .select()
                //这里指定Controller扫描包路径
                .apis(RequestHandlerSelectors.basePackage(basePackage))
                .paths(PathSelectors.any())
                .build();
    }

    private ApiInfo apiInfo() {
        return new ApiInfoBuilder().title(title).description(description)
                .version(version)
                .termsOfServiceUrl("http://localhost:8080")
                .contact(new Contact("Hans", "", "hyman213@163.com"))
                .build();
    }

}

API文档访问地址：http://127.0.0.1:8080/doc.html

Swagger2 注解​

主要注解​

  • @Api - 定义接口分组名称
  • @ApiImplicitParam - 单个参数注释
  • @ApiImplicitParams - 多个参数注释
  • @ApiModel - 实体类定义
  • @ApiModelProperty - 实体属性定义
  • @ApiOperation - 接口定义
  • @ApiParam - 参数注释
  • @ApiResponse - 响应码
  • @ApiResponses - 多个响应码
  • @ApiIgnore - 使用该注解忽略这个API
  • @ApiError - 发生错误返回的信息

代码示例​

@ApiIgnore - Swagger 文档不会显示拥有该注解的接口

}

方法相关注解​

@ApiOperation - 对接口描述

@ApiParam - 对接口参数描述

@ApiIgnore - Swagger 文档不会显示拥有该注解的接口

@ApiImplicitParams - 用于描述接口的非对象参数集

@ApiImplicitParam - 用于描述接口的非对象参数，一般与 @ApiImplicitParams 组合使用

@ApiResponse - 响应码, 一般用于异常码的申明

@ApiResponses - 多个响应码

@RestController
@Api(tags = "用户管理相关接口")
@RequestMapping("/user")
public class UserController {

    @PostMapping("/")
    @ApiOperation("添加用户")
    @ApiImplicitParams({
            @ApiImplicitParam(name = "username", value = "用户名", defaultValue = "李四"),
            @ApiImplicitParam(name = "address", value = "用户地址", defaultValue = "深圳", required = true)
    })
    @ApiResponses({
        @ApiResponse(code = USER_NAME_DUPLICATED, message = "用户名重复")  
    })
    public RespBean addUser(String username, @RequestParam(required = true) String address) {
        return new RespBean();
    }
    
    @GetMapping("/{id}")
    // 通过hidden=true，可隐藏该接口
    @ApiOperation("查询用户")
    @ApiResponses({
        @ApiResponse(code = USER_NOT_FOUND, message = "用户不存在")  
    })
    @ApiImplicitParam(name = "id", value = "用户id", defaultValue = "99", required = true)
    public User getUserById(@ApiParam(value = "用户ID", required = true)  
                            @PathVariable(value = "id") String userId) {
        User user = new User();
        user.setId(id);
        return user;
    }
    
    @PutMapping("/{id}")
    @ApiResponses({  
        @ApiResponse(code = USER_NOT_FOUND, message = "用户不存在")  
    })
    @ApiOperation("更新用户")
    public User updateUserById(@ApiParam(value = "用户ID", required = true)  
                                @PathVariable(value = "id") String userId,
                                @RequestBody User user) {
        return user;
    }
}

注意

  • 指定接口的method（GET/POST...），否则每个method方式都会有接口

参数Model相关注解​

@ApiModel - 相关实体的描述，当value名冲突时，在显示时可能会存在错乱的现象，建议使用默认的value，添加description描述

@ApiModelProperty - 实体属性的相关描述，默认指定value即可，常用的改用 required，dataType，

@ApiModel(description = "用户信息")
public class User {
    @ApiModelProperty(value = "用户id")
    private Integer id;
    @ApiModelProperty(value = "用户名")
    private String username;
    @ApiModelProperty(value = "用户地址")
    private String address;
    //getter/setter
}
