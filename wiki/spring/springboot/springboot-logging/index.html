SpringBoot日志配置

日志相关的官方文档地址：https://docs.spring.io/spring-boot/docs/current/reference/html/features.html#features.logging

日志相关可配置项：https://docs.spring.io/spring-boot/docs/current/reference/html/application-properties.html

默认日志配置，最简单的日志配置​

SpringBoot 默认日志级别及输出格式​

SpringBoot 默认选择的是logback来记录日志。application.yml 中不加任何日志相关的配置，默认只输出INFO级别及以上的日志

2022-10-19 22:28:40.133  INFO 25728 --- [nio-8082-exec-1] c.h.tech.springboot.LoggingApplication   : -----> info <-----
2022-10-19 22:28:40.134  WARN 25728 --- [nio-8082-exec-1] c.h.tech.springboot.LoggingApplication   : -----> warn <-----
2022-10-19 22:28:40.134 ERROR 25728 --- [nio-8082-exec-1] c.h.tech.springboot.LoggingApplication   : -----> error <-----

实现一个极简的日志配置​

logging:
  level:
    com.hans: DEBUG
  pattern:
    console: '%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n'
    file: '%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n'
  file:
    path: ./logs
    name: ${logging.file.path}/${spring.application.name}.log
  logback:
    rollingpolicy:
      max-file-size: 50MB
      max-history: 7
2022-10-19 22:40:26.822 [http-nio-8082-exec-1] DEBUG c.h.t.springboot.LoggingApplication - -----> debug <-----
2022-10-19 22:40:26.823 [http-nio-8082-exec-1] INFO  c.h.t.springboot.LoggingApplication - -----> info <-----
2022-10-19 22:40:26.823 [http-nio-8082-exec-1] WARN  c.h.t.springboot.LoggingApplication - -----> warn <-----
2022-10-19 22:40:26.823 [http-nio-8082-exec-1] ERROR c.h.t.springboot.LoggingApplication - -----> error <-----

踩过的坑：当我向下面这样配置的时候，logging.file.path 没有生效, 应该是path和name不能同时使用。官方描述是：

you need to set a logging.file.name or logging.file.path property

logging:
  level:
    com.hans: DEBUG
  file:
    path: ./logs
    name: ${spring.application.name}.log

logback.xml 配置​

logback默认配置文件名为logback-spring.xml，也可以在 application.yml 中通过logging.config指定自己的配置文件路径

Logback 简明使用手册: https://my.oschina.net/u/566591/blog/651866

使用自定义logback自定义配置，则在application.yml加下面配置

logging:
  config: classpath:my-logback.xml
  level:
    root: INFO
    com.hans: DEBUG

logback.xml 配置示例

<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <!-- include默认的配置 -->
    <include resource="org/springframework/boot/logging/logback/defaults.xml"/>

    <!-- springProperty 可以读取application.yml中的配置 -->
    <springProperty scope="context" name="APP_NAME" source="spring.application.name"/>
    <springProperty scope="context" name="PROFILE" source="spring.profiles.active" defaultValue="dev"/>
    <springProperty scope="context" name="LOG_HOME" source="logging.file.path" defaultValue="logs"/>
    <!-- 日志保留天数 -->
    <springProperty scope="context" name="FILE_MAX_HISTORY" source="logging.logback.rollingpolicy.max-history"
                    defaultValue="7"/>
    <springProperty scope="context" name="LOG_ASYNC_QUEUE_SIZE" source="logging.async.queue-size"
                    defaultValue="512"/>
    <!-- 是否写入本地json文件 -->
    <springProperty scope="context" name="JSON_LOG_ENABLE" source="logging.file.json.enable" defaultValue="true"/>

    <!-- 定义变量 -->
    <property name="LOG_FILE" value="${LOG_HOME}/${APP_NAME}"/>
    <property name="LOG_PATTERN"
              value="[%d{yyyy-MM-dd HH:mm:ss.SSS}] [%thread] [${APP_NAME:-},%X{traceId:-},%X{spanId:-},%X{spanExportable:-}] [%-5level] [%-36.36logger{36}:%-4.4line] - %msg%n"/>

    <!-- 在特定profile下的配置 -->
    <springProfile name="product">
        <!-- configuration to be enabled when the "staging" profile is active -->
    </springProfile>

    <!-- 控制台输出 -->
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <!-- encoders are assigned the type
           ch.qos.logback.classic.encoder.PatternLayoutEncoder by default -->
        <encoder class="ch.qos.logback.classic.encoder.PatternLayoutEncoder">
            <pattern>${LOG_PATTERN}</pattern>
            <charset>utf8</charset>
        </encoder>
    </appender>

    <!-- 输出到文件 -->
    <appender name="LOG_FILE"
              class="ch.qos.logback.core.rolling.RollingFileAppender">
        <filter class="ch.qos.logback.classic.filter.ThresholdFilter">
            <level>INFO</level>
        </filter>
        <file>${LOG_FILE}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_FILE}.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>${FILE_MAX_HISTORY}</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>${LOG_PATTERN}</pattern>
            <charset>utf8</charset>
        </encoder>
    </appender>

    <!-- 输出自定义JSON日志，用于filebeat采集-->
    <appender name="JSON_FILE"
              class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_FILE}.json</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_FILE}.%d{yyyy-MM-dd}.json</fileNamePattern>
            <maxHistory>${FILE_MAX_HISTORY}</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>${LOG_PATTERN}</pattern>
            <charset>utf8</charset>
        </encoder>
        <!--        <encoder class="ch.qos.logback.core.encoder.LayoutWrappingEncoder">-->
        <!--            &lt;!&ndash; 替换class 为自定义的EventLayout &ndash;&gt;-->
        <!--            <layout class="ch.qos.logback.classic.encoder.PatternLayoutEncoder">-->
        <!--                &lt;!&ndash; 传参 &ndash;&gt;-->
        <!--                <source>${APP_NAME}</source>-->
        <!--                <tag>${PROFILE}</tag>-->
        <!--            </layout>-->
        <!--        </encoder>-->
    </appender>

    <!-- 异步写入自定义json日志 -->
    <appender name="ASYNC-LOG-FILE" class="ch.qos.logback.classic.AsyncAppender">
        <!-- 异步写入日志队列的深度-->
        <queueSize>${LOG_ASYNC_QUEUE_SIZE}</queueSize>
        <discardingThreshold>0</discardingThreshold>
        <neverBlock>true</neverBlock>
        <appender-ref ref="JSON_FILE"/>
    </appender>

    <root level="INFO">
        <!--生产环境不输出控制台日志 -->
        <if condition='!property("PROFILE").equalsIgnoreCase("product")'>
            <then>
                <appender-ref ref="STDOUT"/>
            </then>
        </if>
        <!-- 总是输出INFO、ERROR日志 -->
        <appender-ref ref="LOG_FILE"/>
        <!-- 异步写入json文件 -->
        <!-- 需要引入Janino依赖 -->
        <if condition='property("JSON_LOG_ENABLE").equalsIgnoreCase("true")'>
            <then>
                <appender-ref ref="ASYNC-LOG-FILE"/>
            </then>
        </if>
    </root>

    <!--    &lt;!&ndash; 生产环境不输出控制台日志 &ndash;&gt;-->
    <!--    <springProfile name="test,dev,local,default">-->
    <!--        <logger name="com.hans.tech.springboot.LoggingApplication">-->
    <!--            <appender-ref ref="STDOUT"/>-->
    <!--        </logger>-->
    <!--    </springProfile>-->

</configuration>

踩过的坑：

  1. 报错：Could not find Janino library on the class path

在logback.xml中使用了条件语句，需要引入依赖 Janino 

<properties>
    <logback-janino.version>3.0.7</logback-janino.version>
</properties>

<dependency>
    <groupId>org.codehaus.janino</groupId>
    <artifactId>janino</artifactId>
    <version>${logback-janino.version}</version>
</dependency>
