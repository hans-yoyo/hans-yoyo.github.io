SpringBoot 整合 JPA

简介​

JPA 与 Hibernate、Spring Data JPA 的关系​

JPA 全称 Java Persistence API。JPA 仅仅是一种规范，也就是说 JPA 仅仅定义了一些接口，而 Hibernate 就是实现了 JPA 接口的ORM框架。

Spirng Data JPA 是spring提供的一套简化JPA开发的框架，按照约定好的【方法命名规则】写dao层接口，就可以在不写接口实现的情况下，实现对数据库的访问和操作。同时提供了很多除了CRUD之外的功能，如分页、排序、复杂查询等等。

Spring Data JPA 可以理解为 JPA 规范的再次封装抽象，底层还是使用了 Hibernate 的 JPA 技术实现。

SpringBoot 整合 SpringData JPA​

快速实现​

  1. 引入依赖
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
  1. 定义Entity
@Data
@Entity
@Table(name = "t_employee")
public class Employee {

    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private Long id;

    private String firstName;

    private String lastName;

    public Employee() {
    }

    public Employee(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }
}
  1. 定义 Repository
public interface EmployeeRepository extends JpaRepository<Employee, Long>, JpaSpecificationExecutor<Employee> {
}
  1. 基本配置
spring:
  jpa:
    hibernate:
      ddl-auto: none
    show-sql: true
    database-platform: org.hibernate.dialect.MySQL5InnoDBDialect
    open-in-view: false

Spring Data Repository​

SpringBoot JPA 提供了三种 Repositories

  1. CrudRepository: 提供了基本的 增、删、改、查操作，比如：findOne(), findAll(), save(), delete()
  2. PagingAndSortingRepository：继承 CrudRepository，扩展了排序和分页查询操作
  3. JpaRepository：提供更为丰富的操作

还有一个比较常用的接口 JpaSpecificationExecutor, 可用于动态生成query，提供了一个高级的入口和结构，通过这个入口可以使用底层JPA的Criteria的所有方法。

JPA中常用注解​

对象与数据库 映射​

  1. @Entity：标记当前类是一个 Entity
import javax.persistence.Entity;

@Entity
public class Employee implements Serializable {
}
  1. @Table：标记该 Entity 映射的表信息，默认是实体名，可用 name 指定表名
import javax.persistence.Entity;
import javax.persistence.Table;

@Entity
@Table(name = "t_employee")
public class Employee implements Serializable {
}
  1. @Column：如果字段名与表字段名相同，可省略该注解，name 指明对应的表字段名
@Column(name = "employee_name")
private String employeeName;
  1. Id：标记 Entity 的主键，必须
  2. @GeneratedValue：主键的生成逻辑
@Data
@Entity
@Table(name = "t_employee")
public class Employee {

    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private Long id;
}

JPA提供的四种标准生成策略为：AUTO、IDENTITY、SEQUENCE、TABLE。

生成策略描述
AUTO自动选择一个最适合底层数据库的主键生成策略。如MySQL会自动对应auto increment。这个是默认选项，即如果只写@GeneratedValue，等价于@GeneratedValue(strategy=GenerationType.AUTO)。
IDENTITY表自增长字段。
SEQUENCE通过序列产生主键。
TABLE通过表产生主键，框架借由表模拟序列产生主键，使用该策略可以使应用更易于数据库移植。不同的JPA实现商生成的表名是不同的，如 OpenJPA生成openjpa_sequence_table表，Hibernate生成一个hibernate_sequences表，而TopLink则生成sequence表。这些表都具有一个序列名和对应值两个字段，如SEQ_NAME和SEQ_COUNT。

自定义ID生成器, CustomIDGenerator 需要继承 IdentityGenerator类，实现 generate() 方法

@Id
@GeneratedValue(strategy = GenerationType.AUTO, generator = "custom-id")
@GenericGenerator(name = "custom-id", strategy = "com.hans....CustomIDGenerator")
private Long id;
  1. @Version：标记字段为版本控制字段，实现乐观锁
@Column
@Version
private Long updateVersion;
  1. @Transient：标记 non-persistent 字段，不会存储到数据库
@Transient
Private int employeePhone;
  1. @Lob：大对象字段
@Lob
public String getEmployeeAddress() {
    return employeeAddress;
}

其他注解​

  1. @NoRepositoryBean

JPA不会创建对应接口的bean对象，一般用于定义通用的 Repository 接口，JpaRepository 就使用了该注解

@NoRepositoryBean
@Transactional
public interface BaseRepository<T, ID extends Serializable>
        extends JpaRepository<T, ID>, JpaSpecificationExecutor<T> {
}
  1. @MappedSuperclass

标注为 @MappedSuperclass 的类将不是一个完整的实体类，不会映射到数据库表，但是它的属性都将映射到其子类的数据库字段中。标注为 @MappedSuperclass 的类不能再标注 @Entity 或 @Table 注解

  1. @EntityListeners

监听JPA实体持久化，可以对实体的属性信息在增删改前后进行监听

示例：https://blog.csdn.net/lxh_worldpeace/article/details/105967763

  1. @DynamicInsert

表示 insert 对象时，字段值是 null 的字段，不会被加入到 insert 语句中。默认: true

  1. @DynamicUpdate

For updating, should this entity use dynamic sql generation where only changed columns get referenced in the prepared sql statement 在更新时，使用动态SQL，只会将字段值发生变化的字段放入 prepared sql statement

update 对象时，如果字段的值没有发生变化，不会被加入到 update 语句中。

在字段比较多，更新频繁，且不是每次所有字段都需要更新的实体，可以标注该注解。

JPA配置详解​

spring:
  jpa:
    hibernate:
      ddl-auto: none
    show-sql: true
    database-platform: org.hibernate.dialect.MySQL5InnoDBDialect
    open-in-view: false
    # 以下配置不建议配置，只在特殊场景下使用
    # 优化批量插入
    # https://github.com/amrutprabhu/spring-boot-jpa-bulk-insert-performance
    properties:
      hibernate:
        format_sql: true
        jdbc:
          batch_size: 500
spring.jpa.hibernate.ddl-auto值说明
  • create: 服务程序重启后，加载hibernate时都会删除上一次服务生成的表，然后根据服务程序中的entity类再重新生成表，这个值慎用，会导致数据库中原表数据丢失。
  • create-drop ：服务服务程序重启后，加载hibernate时根据entity类生成表，当sessionFactory关闭时，创建的表就自动删除。
  • update：默认常用属性，第一次加载hibernate时根据entity类会自动建立表结构，后面服务程序重启时，加载hibernate会根据entity类自动更新表结构，如果表结构改变了，但是表行仍然存在，不会删除以前的行（对于表结构行只增不减）。
  • validate ：服务程序重启后，每次加载hibernate时，验证创建数据库表结构，只会和数据库中的表进行比较，如果不同，就会报错。不会创建新表，但是会插入新值。
  • none : 什么也不做, 现在一般使用 flyway 管理数据表，所以使用该配置

更多配置：https://docs.spring.io/spring-boot/docs/current/reference/html/application-properties.html

public interface ICommentCount {
    Integer getYearComment();
    Long getTotalComment();
}

// Repository
@Query("SELECT c.year AS yearComment, COUNT(c.year) AS totalComment "
  + "FROM Comment AS c GROUP BY c.year ORDER BY c.year DESC")

构建动态SQL查询​

Specification<Paper> specification = (root, criteriaQuery, criteriaBuilder) -> {
    List<Predicate> predicates = Lists.newArrayList();
    if (subject != null) {
        predicates.add(criteriaBuilder.equal(root.get("subject"), subject));
    }

    if (!CollectionUtils.isEmpty(grades)) {
        if (grades.size() == 1) {
            predicates.add(criteriaBuilder.equal(root.get("grade"), grades.get(0)));
        } else {
            predicates.add(criteriaBuilder.in(root.get("grade")).value(grades));
        }
    }

    if (edition != null) {
        predicates.add(criteriaBuilder.equal(root.get("edition"), edition));
    }

    if (term != null) {
        predicates.add(criteriaBuilder.equal(root.get("term"), term));
    }

    if (type != null) {
        predicates.add(criteriaBuilder.equal(root.get("type"), type));
    }

    if (StringUtils.isNotEmpty(keywords)) {
        predicates.add(criteriaBuilder.like(root.get("name"), "%" + keywords.trim() + "%"));
    }

    return criteriaBuilder.and(predicates.toArray(new Predicate[predicates.size()]));
};

Page<Paper> pageQueryResult = paperRepository.findAll(specification, pageRequest);

Spring JPA Audit​

  1. 启动类添加注解 @EnableJpaAuditing
@SpringBootApplication
@EnableJpaAuditing
public class JpaApplication {

    public static void main(String[] args) {
        SpringApplication.run(JpaApplication.class, args);
    }
    
}
  1. 定义 BaseEntity，加上注解 @EntityListeners(AuditingEntityListener.class)
@MappedSuperclass
@Data
@EntityListeners(AuditingEntityListener.class)
public class BaseEntity implements Serializable {

    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private Long id;

    /**
     * 创建时间，由JPA管理
     */
    @Column(nullable = false, updatable = false)
    @CreatedDate
    private Long createTime;
    /**
     * 创建人，由JPA管理
     */
    @Column(updatable = false)
    @CreatedBy
    private String createBy;
    /**
     * 最后更新时间，由JPA管理
     */
    @Column(nullable = false)
    @LastModifiedDate
    private Long updateTime;
    /**
     * 最后更新人，由JPA管理
     */
    @Column
    @LastModifiedBy
    private String updateBy;
    /**
     * 乐观锁
     */
    @Column
    @Version
    private Long updateVersion;

}
  1. 实现 AuditorAware 接口，实现 getCurrentAuditor() 方法
import org.springframework.context.annotation.Configuration;
import org.springframework.data.domain.AuditorAware;
import org.springframework.security.core.context.SecurityContext;
import org.springframework.security.core.context.SecurityContextHolder;

@Configuration
public class UserIDAuditorBean implements AuditorAware<Long> {
    @Override
    public Long getCurrentAuditor() {
        SecurityContext ctx = SecurityContextHolder.getContext();
        if (ctx == null) {
            return null;
        }
        if (ctx.getAuthentication() == null) {
            return null;
        }
        if (ctx.getAuthentication().getPrincipal() == null) {
            return null;
        }
        Object principal = ctx.getAuthentication().getPrincipal();
        if (principal.getClass().isAssignableFrom(Long.class)) {
            return (Long) principal;
        } else {
            return null;
        }
    }
}

JPA几个注意点​

默认先select，再update​

save方法在更新DB时一个最大的特点是：默认每次都会先去查一遍DB，对查出来的DB数据与要保存的数据进行比较，看是否有变化，若有变化，才会将数据持久化至DB（执行数据的update），否则，就不会进行数据的持久化。

默认更新所有的字段​

save方式下的更新数据，会默认更新该条记录的所有字段，即使你原本只更改了一个字段值，但最后更新DB时，JPA依旧会对该条记录的所有字段进行更新

JPA中实体字段与数据库不一致的影响​

一、通过JSQL，即通过方法名自动生成SQL

List<TRecord> findAllByIdIsAfter(Long startId);
  1. 实体字段少于数据库字段，不影响查询，生成的SQL只查询实体类中定义的字段
  2. 实体字段多余数据库字段，查询报错，生成的SQL中带有数据库不存在的字段

这也验证了在生产环境数据库中，新增字段，不会影响原有服务。

二、通过编写原生的SQL，

  1. 实体字段少于数据库字段，不影响查询
  2. 实体字段多余数据库字段，查询报错，生成的SQL中带有数据库不存在的字段
@Query(nativeQuery = true, value = "SELECT * FROM t_record WHERE id>?1 ORDER BY ID ASC LIMIT ?2")  
List<TRecord> findAllByIdIsAfterLimit(Long startId, Integer batchSize);

结论：实体类的字段不能多余数据库中列，一旦实体类中定义的字段，在数据库表中不存在，则会导致报错

JPA save() 与 saveAndFlush() 的区别​

save() 只将数据更新到了内存，而没有及时的更新到DB saveAndFlush()，数据同步更新到内存和DB。

一般的操作，使用save就可以了，如果在一个事务中，后一个操作依赖上一个DB的更新，这个时候就需要使用 saveAndFlush

save是CrudRepository下的。saveAndFlush是JpaRepository下的。 一般情况下save就足够了，但是如果在同一个transaction里，比如你在方法上标了@Transactional，这时你想先保存Order，然后直接在这个Transaction里保存OrderItem，但是OrderItem需要新建的Order的ID，这时候保存Order就可以用saveAndFlush，你的Entity就会有保存在数据里的id了。
