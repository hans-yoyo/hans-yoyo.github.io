数据库索引失效的几种场景

[!NOTE] 索引失效的几种情况

  1. 字段类型不匹配
  2. 被索引字段使用了计算表达式
  3. 被索引字段使用了内置函数
  4. 使用了靠右(%xxx)模糊查询
  5. or 分割的条件，左边的条件存在索引，右边的条件不存在索引
  6. 子查询 IN、NOT IN 会导致索引失效
CREATE TABLE IF NOT EXISTS `user_info` (
 `id` bigint(20) NOT NULL AUTO_INCREMENT,
 `name` VARCHAR(20) NOT NULL,
 `gender` int(2) NOT NULL,
 `age` int(10) NOT NULL,
 `phone_number` VARCHAR(20) NOT NULL,
 `address` VARCHAR(40) NOT NULL,
  PRIMARY KEY ( `id` ),
  KEY `idx_name`(`name`),
  KEY `idx_phone`(`phone_number`),
  KEY `idx_age_address`(`age`,`address`)
)ENGINE=InnoDB DEFAULT CHARSET=utf8;

字段类型不匹配​

where 条件字段类型与实际表中字段类型不匹配的时候, Mysql会进行隐式转换，隐式转换会用到内置函数，导致索引失效

EXPLAIN SELECT * FROM user_info WHERE phone_number = 13688886666;

EXPLAIN SELECT * FROM user_info WHERE phone_number = '13688886666';

被索引字段使用了计算表达式​

EXPLAIN SELECT * FROM user_info WHERE age + 2 = 30;

被索引字段使用了内置函数​

EXPLAIN SELECT * FROM user_info WHERE LEFT(phone_number, 3) = '136';

使用了靠右(%xxx)模糊查询​

EXPLAIN SELECT * FROM user_info WHERE name LIKE 'xiao%';

EXPLAIN SELECT * FROM user_info WHERE name LIKE '%xiao%';

索引字段不是联合索引字段的最左字段​

EXPLAIN SELECT * FROM user_info WHERE address = '武汉市';

EXPLAIN SELECT * FROM user_info WHERE age = 18;

or 分割的条件，左边的条件存在索引，右边的条件不存在索引​

子查询 IN、NOT IN 会导致索引失效​
